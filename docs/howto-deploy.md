# How to deploy and manage redirects

How a change reaches production, and how to add a redirect. The site is static
and hosted on Netlify; deploys are driven by pushes to GitHub.

## How a deploy happens

```
edit content/code ──> commit ──> push to GitHub ──> Netlify builds ──> live
        (locally or via Pages CMS)                   (npm run build)
```

There is no manual deploy step. Netlify watches the repo, runs the build on
every push, and publishes the output. Pages CMS edits are just commits, so they
follow the same path. See
[Edit content with Pages CMS](./howto-edit-content-with-pages-cms.md).

## Build locally before pushing

The same build Netlify runs:

```bash
npm run build     # outputs static files to dist/
npm run preview   # serve dist/ locally to check the production build
```

Running `npm run build` locally is the fastest way to catch a content schema
error or a broken MDX body before it fails on Netlify.

## Verification

- Watch the Netlify deploy for the pushed commit finish green.
- Load the affected page on `https://joshux.com`.
- For a new case study, also check `https://joshux.com/work/<slug>/` and the
  listing at `/case-studies`.

## Adding a redirect

Legacy and renamed URLs are handled by `public/_redirects` (Netlify's redirect
file). Each line is `from  to  status`.

```
/projects/axs-case-study/   /work/axs-event-entry-case-study   301
/work/ribbon-case-study     /work/ribbon-legalzoom-case-study  301
```

To add one: append a line, commit, and push. Use `301` for permanent moves so
search engines transfer ranking. Existing entries cover the old `/projects/*`
paths, several renamed `/work/*` slugs, and legacy `.html` URLs.

## Headers

`public/_headers` sets response headers. Today it adds `X-Robots-Tag: noindex,
nofollow` to the resume PDF so it is not indexed. Add header rules the same way:
a path followed by indented header lines.

## What is and is not indexed

Three controls decide search visibility:

- **`/robots.txt`** is generated by `src/pages/robots.txt.ts`. It allows
  everything except `/joshwright-resume.pdf` and points to the sitemap.
- **The sitemap** (`sitemap-index.xml`) is generated by `@astrojs/sitemap` at
  build. `astro.config.mjs` filters the makerspace case study out of it, so that
  page builds and is reachable but is not listed for crawlers.
- **`hidden: true`** on a case study adds `noindex` to that page and removes it
  from listings. See [Content schema](./reference-content-schema.md).

## Troubleshooting

- **Netlify build fails.** Reproduce locally with `npm run build`. The error is
  usually a content schema mismatch or an MDX syntax error; the message names
  the file.
- **A redirect does not fire.** `_redirects` only applies on Netlify, not in
  `npm run dev`. Test it on a deploy preview or production. Check the status
  code and that the `from` path matches exactly (including the trailing slash).
- **A page you expected is missing from search.** Check whether it is `hidden`,
  filtered from the sitemap, or `*.bak.mdx` (which never builds at all).
